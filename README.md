## Featured Projects

<details>
<summary><strong>🔬 Multimodal Healthcare AI & Diagnostic Suite</strong></summary>

<br/>

> An AI-powered diagnostic support system that fuses medical imaging data, structured lab reports, and clinical notes to surface disease-risk predictions across multiple modalities — integrating CNNs for image analysis with NLP for conversational diagnostic guidance.

<br/>

| Attribute | Detail |
|:---|:---|
| **Stack** | Python · CNNs · Machine Learning · NLP · Gemini API |
| **Scale** | Multi-modal inputs: MRI scans, X-rays, blood reports, symptom data |
| **Performance** | CNN-based image models with competitive accuracy on medical imaging benchmarks |
| **Security** | Controlled data handling · Diagnostic output validation layer |
| **Impact** | Guides users through risk assessment via NLP-assisted conversational interface |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-7C3AED?style=flat-square&logo=github&logoColor=white)](https://github.com/Janhvi232004) |

<br/>

The system is built around three tightly integrated components. A **CNN-based vision module** processes MRI scans and X-ray images through a domain-adapted convolutional architecture, trained for preliminary disease-risk classification. A **machine learning layer** ingests structured lab and blood report data to produce tabular risk signals. An **NLP-assisted interface** combines these outputs with a conversational front-end built using Gemini API, guiding users through risk assessments and surfacing actionable diagnostic insights in plain language. The architecture is designed to be modular — each inference component operates independently and contributes to a unified risk profile, with integration hooks for AI-driven healthcare workflow systems.

<details>
<summary><strong>📝 Automated Text Summarisation Tool</strong></summary>

<br/>

> An NLP-based document intelligence system capable of producing concise, contextually faithful summaries from long-form textual documents — supporting both extractive and abstractive summarisation techniques across multiple document formats.

<br/>

| Attribute | Detail |
|:---|:---|
| **Stack** | Python · Gemini API · NLP · HuggingFace · Multi-format document parsing |
| **Scale** | Handles research papers, articles, reports, and multi-format document inputs |
| **Performance** | Preserves contextual accuracy through hybrid extractive + abstractive strategy |
| **Security** | Input sanitisation · API key management · Rate-limited inference |
| **Impact** | Reduces document review time significantly · Supports multiple output verbosity levels |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-7C3AED?style=flat-square&logo=github&logoColor=white)](https://github.com/Janhvi232004) |

<br/>

The pipeline begins with a **pre-processing layer** that handles format normalisation across PDF, DOCX, HTML, and plain-text inputs, followed by intelligent chunking calibrated to document structure. The **Gemini API-powered summarisation layer** applies dynamically selected prompt templates — extractive for factual density, abstractive for readability — based on document type. A **post-processing layer** enforces length calibration, structural consistency, and output formatting. The system supports multi-document ingestion for comparative summarisation and provides efficient information retrieval across large corpora, enabling users to query summarised content without re-processing source documents.

<br/>

</details>

<details>
<summary><strong>🤖 Custom Knowledge Base Question-Answering System</strong></summary>

<br/>

> A RAG-powered question-answering system that grounds GPT-4 responses in private document corpora using vector embeddings and semantic retrieval — eliminating hallucinations, reducing latency, and delivering citation-traceable answers from domain-specific knowledge bases.

<br/>

| Attribute | Detail |
|:---|:---|
| **Stack** | Python · GPT-4 · RAG · Vector Embeddings · Semantic Search |
| **Scale** | Indexes and queries large private document corpora |
| **Performance** | Hallucination-reduced responses · Optimised prompt workflows for domain specificity |
| **Security** | Private data isolation · Query logging · Input validation |
| **Impact** | Enables organisations and researchers to query proprietary knowledge with precision |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-7C3AED?style=flat-square&logo=github&logoColor=white)](https://github.com/Janhvi232004) |

<br/>

The system implements a **full RAG pipeline**: documents are parsed, chunked, and embedded into a vector store using dense embedding models. At query time, semantic search retrieves the top-k most relevant passages, which are passed alongside the user's question to GPT-4 for grounded response generation. **Vector embedding pipelines** are optimised for contextual relevance — reducing surface-level keyword matching in favour of semantic proximity. A **prompt engineering layer** structures the GPT-4 context window with strict grounding instructions, ensuring every answer is traceable to a retrieved passage. Domain-specific prompt templates are maintained per knowledge base, enabling accurate, jargon-aware responses across specialised corpora.

<br/>

</details>
